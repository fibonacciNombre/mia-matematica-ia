### Contenido Matemático: Conjuntos y Relaciones

**Conceptos Fundamentales de Conjuntos**

- Conjunto: colección de objetos/elementos que se pueden enumerar directamente o caracterizar por una propiedad
- Igualdad de conjuntos requiere verificar doble inclusión (cada elemento de uno pertenece al otro y viceversa)
- Conjunto vacío: no tiene elementos, es subconjunto de cualquier conjunto
- Cardinalidad: número de elementos de un conjunto (finito o infinito)
- Importante: cardinalidad infinita de números pares ≠ cardinalidad infinita de números reales

**Comparación y Operaciones entre Conjuntos**

- Pertenencia (∈): relación entre elemento y conjunto
- Inclusión (⊆): relación entre conjuntos (subconjunto)
- Error común: no confundir pertenencia con inclusión
- Operaciones binarias: unión, intersección, diferencia
- Complemento: elementos que no pertenecen al subconjunto respecto al conjunto de referencia

**Aplicación en Machine Learning**

- Conjunto de referencia = base de datos o data completa
- Subconjunto para entrenamiento del modelo/función con calibración de parámetros
- Complemento = data para verificar si modelo entrenado genera buenas previsiones

**Propiedades de Operaciones**

- Propiedad conmutativa: orden no afecta resultado (unión e intersección)
- Propiedad asociativa: permite operar múltiples conjuntos sin importar agrupación
- Propiedad distributiva: intersección sobre unión y viceversa
- Ley de De Morgan: complemento se distribuye cambiando operación (intersección ↔ unión)
- Principio de inclusión-exclusión para cardinalidad

**Producto Cartesiano**

- Conjunto de parejas ordenadas donde primera entrada es del primer conjunto, segunda del segundo conjunto
- Orden importa según contexto (geometría, matrices)
- Cardinalidad del producto cartesiano = producto de cardinalidades
- Generalización a n-tuplas para productos de múltiples conjuntos
- Fundamental en ML: bases de datos, espacios vectoriales, concepto de dimensión

**Conjuntos Numéricos**

- Naturales (ℕ): números de conteo, base para otros conjuntos
- Enteros (ℤ): incluyen negativos (opuestos aditivos) necesarios para operación de adición
- Racionales (ℚ): cocientes de enteros, representación decimal exacta, periódica o periódica mixta
- Irracionales: decimales que no son exactos ni periódicos (√2, π, e)
- Reales (ℝ): unión de racionales e irracionales, usado en casi todos los modelos de IA
- Complejos (ℂ): extensión de reales con unidad imaginaria (i² = -1), usados en acústica, señales, circuitos eléctricos y algunos modelos recientes de IA
- Reales están estrictamente contenidos en complejos
- Conjugado de número complejo: cambio de signo en parte imaginaria

### Relaciones entre Conjuntos

**Definición y Representación**

- Relación: subconjunto del producto cartesiano entre dos conjuntos
- Notación: xRy indica que (x,y) pertenece a la relación R
- Representación geométrica posible en sistema cartesiano para conjuntos numéricos

**Interpolación y Modelado**

- Búsqueda de función/curva que pase por puntos de la relación
- Más importante que interpolación exacta: observar comportamiento y distribución de datos
- En ML: nube de puntos (scatter) requiere enfoque estadístico-probabilístico para mejor modelo

**Propiedades de Relaciones**

- Reflexiva: cada elemento relacionado consigo mismo (ejemplo: igualdad)
- Antireflexiva: ningún elemento relacionado consigo mismo (ejemplo: menor estricto)
- Simétrica: si xRy entonces yRx (ambas parejas ordenadas presentes)
- Antisimétrica: para elementos diferentes, si xRy entonces no yRx
- Transitiva: si xRy y yRz entonces xRz (elemento intermedio como nexo)

**Tipos Especiales de Relaciones**

- Relación de equivalencia: reflexiva + simétrica + transitiva (ejemplo: igualdad, congruencia geométrica)
- Relación de orden: reflexiva + antisimétrica + transitiva (ejemplo: ≤, inclusión ⊆)
- Relación de orden estricto: antireflexiva + antisimétrica + transitiva (ejemplo: <, inclusión estricta)

**Aplicaciones**

- Congruencia geométrica relevante en etiquetado de imágenes satelitales, fotogrametría, radiografías, tomografías
- Relación de equivalencia genera partición del conjunto (subconjuntos disjuntos cuya unión es el total)
- Clusterización en ML: agrupación de data según relación de equivalencia
- Correspondencia biunívoca: relación de equivalencia ↔ partición del conjunto

### Funciones

**Definición**

- Función: relación especial donde cada elemento del dominio se relaciona con **exactamente uno** del codominio
- Terna que define función: dominio (conjunto 1), codominio (conjunto 2), gráfico (subconjunto del producto cartesiano)
- Importante: misma regla de cálculo con diferente dominio o codominio = funciones diferentes

**Conceptos Asociados**

- Imagen de función y pre-imagen de elemento/subconjunto
- Precaución: pre-imagen ≠ recíproco (opuesto multiplicativo)

**Composición de Funciones**

- Operación fundamental en modelos de IA
- Modelo global = secuencia de subprocesos/submodelos, cada uno representado por una función
- Composición: resultado final = f_n ∘ f_{n-1} ∘ ... ∘ f_1 aplicado a data de entrada
- Mejorar modelo global requiere entender y modificar submodelos componentes
- Propiedad asociativa de composición permite flexibilidad operativa

**Tipos de Funciones**

- Inyectiva, sobreyectiva, biyectiva
- Función biyectiva permite definir función inversa respecto a composición
- Función inversa cumple: f ∘ f^{-1} = id_Y y f^{-1} ∘ f = id_X
- Función identidad: devuelve mismo elemento de entrada sin modificación

### Operaciones Binarias

**Definición**

- Operación binaria interna: función de dos argumentos cuyo resultado pertenece al mismo conjunto
- Dominio = producto cartesiano del conjunto consigo mismo, codominio = mismo conjunto
- Ejemplos: intersección de conjuntos, unión de conjuntos, adición de números
- Caso específico: adición de números reales toma pareja de reales y asocia otro real (suma)
